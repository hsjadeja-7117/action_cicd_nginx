// Study Approach
How to use
these notes.
01
Practice every command
Reading is not enough. Spin up a RHEL or CentOS VM and type every command in these notes. Muscle memory is essential in Linux administration.
02
Understand, don't memorise
Focus on understanding what each command does and why. Use man commandname and command --help to explore options you're unsure about.
03
Always verify your work
After any configuration change, verify it worked. Use systemctl status, df -h, mount -a, or lsblk to confirm changes took effect.
04
Build a lab environment
Use VirtualBox or VMware to create a local RHEL 9 lab. Practice storage tasks, user management, and service configuration in an environment where mistakes are safe.

CH-1
Vim & SSH
Shell basics, remote connections, key auth·Beginner
▶

Shell Basics

  • Bash = Bourne Again Shell
  • ~ = home directory | root = administrator
  • Command syntax: command [option/argument] [target/path]

Getting Help

command --help        # Show all available options
man command           # Display the manual page

Essential Commands

ls              # List directory contents
ls -l           # Long listing format
ls -a           # Include hidden files
ls -t           # Sort by modification time (newest first)
touch file      # Create blank file
mkdir dir       # Create blank directory
rm file         # Remove file
rmdir dir       # Remove blank (empty) directory only
rm -rf dir      # Force remove file or directory — use carefully
cat file        # Read file content
head file       # First 10 lines  |  head -n 5 file = first 5
tail file       # Last 10 lines   |  tail -f file = real-time follow
more file       # Gradually read whole file
echo "text"     # Print text or redirect into file

SSH — Secure Shell

ssh username@hostname_or_ip

Key-Based Authentication

ssh-keygen                              # Generate key pair
ssh-copy-id user@remote_hostname        # Copy public key to remote
ssh user@remote_hostname                # Login without password

SSH-Agent (Cache passphrase)

eval $(ssh-agent)               # Start agent background process
ssh-add .ssh/private_key_name   # Register key with agent
ssh user@host                   # Connect without passphrase prompt

Non-Default Key

ssh-keygen -f .ssh/mykey
ssh-copy-id -i .ssh/mykey.pub user@hostname
ssh -i .ssh/mykey user@hostname

echo — Print & Write Text

echo Hello                      # Print text to terminal
echo "Hello Linux" > file.txt   # Write to file (overwrites)
echo "New line" >> file.txt     # Append to file (keeps existing)

Vim Editor — Essential for Administration

Vim is used to edit configuration files, scripts, and system settings. It has three modes:

Mode Purpose How to Enter
Normal Navigation and commands Default / press Esc
Insert Type and edit text Press i
Command Save, quit, search Press : 
vim filename    # Open file in vim

Essential Vim Commands

Action Command
Enter Insert Mode i
Save :w
Quit :q
Save & Quit :wq
Force Quit (no save) :q!
Delete Line dd
Copy Line yy
Paste p
Search /word

Linux Filesystem Hierarchy

Directory Purpose
/boot Bootloader files and kernel (vmlinuz)
/dev Device files (disk, CPU, keyboard)
/home Home directories for regular users
/root Home directory of root user
/etc Configuration files
/tmp Temporary files (world-writable)
/usr/bin User command binaries
/usr/sbin System administration binaries
/var Variable data — logs, spool, cache

Navigation

cd ..          # Move to parent directory
cd ~           # Move to home directory
cd /etc        # Move to absolute path
pwd            # Print current working directory

File Operations

mkdir dir1                  # Create directory
mkdir -p dir1/dir2/dir3     # Create full nested path
touch filename              # Create or update file

cp source dest              # Copy file
cp -r source_dir dest_dir   # Copy directory (must use -r)

mv file /tmp/               # Move file
mv old.txt new.txt          # Rename file

rm file1                    # Delete file
rm -r dir1                  # Delete directory
rm -rf dir1                 # Force delete — no confirmation
rmdir dir1                  # Remove empty directory only

Links

Feature Soft Link (Symbolic) Hard Link
Command ln -s target linkname ln target linkname
Concept Pointer to a path Mirror — same inode
Original deleted Link breaks Data remains safe
Cross partition Yes No

grep

grep "term" file          # Basic search
grep -i "term" file       # Case insensitive
grep -v "term" file       # Invert match
grep -r "term" folder/    # Recursive
grep -w "word" file       # Whole word only
grep "^root" /etc/passwd  # Lines starting with root
grep "bash$" /etc/passwd  # Lines ending with bash

grep "error" logfile | sort | head    # Filter → sort → top 10
CH-3
Users & Groups
useradd, usermod, sudo, /etc/shadow, chage·Intermediate
▶

User Types

Type UID Range Description
Superuser (root) 0 Full system control
System User 1–999 Used by services/daemons
Regular User 1000–60000 Normal login users

Create & Modify Users

useradd username       # Create new user
passwd username        # Set password

usermod -l newname oldname           # Rename
usermod -aG groupname username       # Add secondary group
usermod -d /new/home -m username     # Change + move home
usermod -s /bin/bash username        # Change shell
usermod -L username                  # Lock account
usermod -U username                  # Unlock account
usermod -e YYYY-MM-DD username       # Set expiry date
⚠️ Always use -a with -G. Without -a, all existing group memberships are overwritten.

SUDO

visudo    # Edit sudoers safely

username ALL=(ALL) ALL           # Grant sudo to user
%groupname ALL=(ALL) ALL         # Grant sudo to group
username ALL=(ALL) NOPASSWD:ALL  # Skip password prompt

Switch Users

su username       # Switch (no env load)
su - username     # Switch with full environment
sudo -i           # Switch to root using your own password

/etc/shadow — Password Storage

Field Meaning
Encrypted Password Hashed password value
Last Password Change Days since Jan 1, 1970
Minimum Age Days before password can be changed
Maximum Age Password validity duration
Warning Days Days before expiry to warn user

Password Policy

chage username       # Interactive policy editor
chage -l username    # List current policy
chage -d 0 username  # Force password change at next login
💡 Always verify changes using id username and chage -l username after modifications.
CH-4
File Access Control — DAC
chmod, chown, SUID, SGID, sticky bit, umask·Intermediate
▶

Permission Meanings

Permission On a File On a Directory
r (read) View file content List directory contents (ls)
w (write) Modify file content Create/delete files inside
x (execute) Run as program/script Enter directory (cd into it)

Symbolic Method

chmod u+x file       # Add execute for user
chmod g-w file       # Remove write from group
chmod o=r file       # Set others to read only
chmod a+x file       # Add execute for all
chmod u=rwx,g=rx,o=r file    # Set all at once

Octal Method

chmod 755 file    # rwxr-xr-x
chmod 644 file    # rw-r--r--
chmod 700 file    # rwx------ (only owner has full access)
chmod 777 file    # rwxrwxrwx

Ownership

chown user file            # Change user ownership
chown user:group file      # Change user and group
chown :group file          # Change group only
chgrp groupname file       # Change group only (alternative)

Special Permissions

# SUID (runs as file owner)
chmod u+s file    # Set SUID
chmod 4744 file   # Set SUID (octal prefix 4)

# SGID
chmod g+s path    # Set SGID
chmod 2744 file   # Set SGID (octal prefix 2)

# Sticky Bit (/tmp uses this)
chmod o+t path    # Set sticky bit
chmod 1744 file   # Set sticky bit (octal prefix 1)
⚠️ In production: avoid 777, use SGID for collaborative directories, use sticky bit for shared writable directories like /tmp.

Default Permissions — umask

umask             # Check current umask
umask 027         # Set temporarily (file max=666, dir max=777)
# User-specific: add to ~/.bashrc
# Global: add to /etc/profile

Processes, Monitoring & Task Scheduling
ps, top, kill, nice, crontab, at, tuned·Intermediate
▶

Job Control

command &     # Start in background
jobs          # List shell jobs
fg %1         # Bring job 1 to foreground
bg %1         # Resume stopped job in background
Ctrl-Z        # Suspend current foreground process
disown %1     # Remove job from shell table

Process Listing

ps                         # Processes in current terminal
ps aux                     # All processes, all users
pgrep firefox              # Find PID by name
pstree                     # Process tree
top                        # Real-time viewer

Signals & Kill

kill <PID>             # SIGTERM — polite stop
kill -9 <PID>          # SIGKILL — immediate force kill
kill -SIGSTOP <PID>    # Suspend process
kill -SIGCONT <PID>    # Resume suspended process
pkill processname       # Kill by name
killall processname     # Kill all with exact name

Load Monitoring

uptime     # Shows: current time, uptime, load averages
lscpu      # CPU count — check CPU(s): field
💡 Formula: Load Average ÷ Number of CPUs. If result > 1 → CPU demand exceeds capacity.

Priority — nice & renice

nice -n 10 command         # Start with lower priority (-20=highest, +19=lowest)
renice -n -5 -p 12345      # Change running process priority

crontab — Recurring Scheduling

# ┌─────── Minute     (0-59)
# │ ┌───── Hour       (0-23)
# │ │ ┌─── DayOfMonth (1-31)
# │ │ │ ┌─ Month      (1-12)
# │ │ │ │ ┌ DayOfWeek (0-7, Sun=0 or 7)
# * * * * * command

crontab -e           # Edit cron jobs
crontab -l           # List cron jobs
crontab -r           # Remove ALL cron jobs

*/2 * * * * echo "every 2 minutes"
0 17-19 * * 3 echo "5-7pm Wednesdays"
0 16 * * 2,4 echo "4pm Tue and Thu"

* * * * * /scripts/backup.sh >> /var/log/backup.log 2>&1

at — One-Time Scheduling

at 07:40              # Schedule at specific time
at now + 5 minutes    # Relative scheduling
atq                   # View pending jobs
atrm 1                # Remove job ID 1

CH-9
Basic Storage & Partitions
fdisk, mkfs, mount, fstab, swap·Intermediate
▶

Check Storage

lsblk           # List block devices and partitions
lsblk -f        # Show filesystem type and UUID
lsblk -fp       # Full path + filesystem type + UUID
df -h           # Disk usage of mounted filesystems
free -h         # RAM and swap usage

Create Partition with fdisk

fdisk /dev/sdb     # Open disk
# n = new partition
# p = primary (Enter for default)
# Enter = accept default first sector
# +2G = set size
# p = print/verify
# w = write and exit
partprobe          # Notify kernel

Format & Mount

mkfs.xfs /dev/sdb1         # Format as XFS
mkfs.ext4 /dev/sdb1        # Format as EXT4

mkdir /mnt/data1
mount /dev/sdb1 /mnt/data1    # Temporary mount
df -h                          # Verify

Permanent Mount — /etc/fstab

vim /etc/fstab
/dev/sdb1  /mnt/data1  ext4  defaults  0  0

systemctl daemon-reload
mount -a        # Test (no output = success)
df -h           # Verify
💡 Use UUID instead of device name in /etc/fstab in production. Get UUID: lsblk -f

Complete Storage Workflow

# 1. Detect disk:      lsblk
# 2. Create partition: fdisk → n → size → w → partprobe
# 3. Format:           mkfs.xfs /dev/sdXN
# 4. Mount point:      mkdir /mnt/data
# 5. Temp mount:       mount /dev/sdXN /mnt/data
# 6. Make permanent:   vim /etc/fstab → add entry
# 7. Test:             systemctl daemon-reload && mount -a
# 8. Verify:           df -h
CH-10
LVM — Logical Volume Manager
PV → VG → LV, extend, reduce, delete·Advanced
▶

LVM Hierarchy

Disk → Partition → PV → VG → LV → Filesystem → Mount Point
💡 Memory code: PPVLMF — Partition → PV → VG → LV → Mount → Filesystem

Part A — Create LVM

# 1. Create LVM partition
fdisk /dev/sdb  →  n → Enter → +1G → t → 8e(MBR) → w
partprobe

# 2. Physical Volume
pvcreate /dev/sdb1
pvdisplay

# 3. Volume Group
vgcreate myvg /dev/sdb1
vgdisplay

# 4. Logical Volume
lvcreate -n mylv -L 5G myvg       # By size
lvcreate -n mylv -l 255 myvg      # By PE count
lvdisplay

# 5. Filesystem
mkfs.xfs /dev/myvg/mylv

# 6. Mount
mkdir /mnt/data
vim /etc/fstab
# /dev/myvg/mylv  /mnt/data  xfs  defaults  0  0
mount -a && df -h

Part B — Extend

lvextend -L +1G /dev/myvg/mylv         # Extend by size
lvextend -L +500M -r /dev/myvg/mylv    # -r also resizes filesystem

xfs_growfs /dev/myvg/mylv              # Resize XFS manually
resize2fs /mnt/data                     # Resize EXT manually

Part C — Reduce (EXT only)

⚠️ XFS cannot be reduced. Only EXT2/3/4 supports reduction. Always backup before reducing.
umount /mnt/data
fsck /dev/myvg/mylv
resize2fs /dev/myvg/mylv 3G
lvreduce -L 3G /dev/myvg/mylv
mount -a

Part D — Delete (Reverse Order)

umount /mnt/data
vim /etc/fstab              # Remove entry
systemctl daemon-reload
lvremove /dev/myvg/mylv
vgremove myvg
pvremove /dev/sdb1
CH-11
Boot Process & systemd Services
systemctl, targets, boot stages, password recovery·Advanced
▶

systemd Service Management

systemctl start httpd              # Start service
systemctl stop httpd               # Stop service
systemctl status httpd             # Check status
systemctl enable httpd --now       # Enable + start immediately
systemctl disable httpd --now      # Disable + stop immediately
systemctl restart httpd            # Stop + start
systemctl reload httpd             # Reload config (no stop)
systemctl mask httpd               # Block from starting at all
systemctl unmask httpd             # Remove block

Targets — System Modes

systemctl get-default                        # Check default
systemctl set-default multi-user.target      # Set CLI default
systemctl set-default graphical.target       # Set GUI default
systemctl isolate multi-user.target          # Switch to CLI now

Linux Boot Sequence

  1. BIOS/UEFI — POST, hardware check, finds bootloader
  2. GRUB2 — Shows boot menu, loads kernel + initramfs
  3. Kernel — Detects hardware, starts systemd (PID 1)
  4. systemd — Reads default target, starts services
  5. Login — CLI or GUI based on target

Root Password Recovery

# 1. Reboot → at GRUB menu press Up/Down to stop countdown
# 2. Press 'e' to edit boot entry
# 3. Find line starting with 'linux', add at end: rd.break
# 4. Press Ctrl+X to boot into emergency shell

mount -o remount,rw /sysroot
chroot /sysroot
passwd
touch /.autorelabel
exit && exit
⚠️ The touch /.autorelabel step is required. Without it the system may fail to boot after a password change.

CH-13
SELinux — Security Enhanced Linux
Contexts, ports, booleans, semanage·Advanced
▶

Status & Modes

sestatus          # Detailed status
getenforce        # Quick mode check (Enforcing/Permissive/Disabled)
setenforce 1      # Enforcing (temporary)
setenforce 0      # Permissive (temporary)
vim /etc/selinux/config    # Permanent change (requires reboot)

View Context Labels

ls -Z filename       # File context
ls -dZ /tmp/         # Directory context
ps axZ               # Process contexts

Change Context — Permanent

# Step 1: Add to SELinux policy database
semanage fcontext -a -t httpd_sys_content_t "/root/demo.html"

# Wildcard for entire directory:
semanage fcontext -a -t httpd_sys_content_t "/webdata(/.*)?"

# Step 2: Apply from database
restorecon -v /root/demo.html     # Single file
restorecon -Rv /webdata/          # Recursive directory

Port Contexts

semanage port -l | grep http        # Filter by type
semanage port -a -t http_port_t -p tcp 82    # Add port
systemctl restart httpd

Troubleshooting & Booleans

journalctl | grep sealert                     # Find denials
getsebool -a | grep http                      # List booleans
setsebool httpd_enable_homedirs on -P         # Set permanently
CH-15
Firewall Management
firewall-cmd, zones, services, ports·Intermediate
▶

Zones

Zone Trust Level
trusted All connections accepted
public Default — untrusted public network
dmz Limited access, for server exposure
block All connections rejected
drop All connections silently dropped
💡 Pattern: --get-... view | --add-... allow | --remove-... deny 
firewall-cmd --get-default-zone                     # View default zone
firewall-cmd --set-default-zone=public              # Set default zone

firewall-cmd --add-service=http --permanent
firewall-cmd --remove-service=http --permanent

firewall-cmd --add-port=8080/tcp --permanent
firewall-cmd --remove-port=8080/tcp --permanent

firewall-cmd --add-source=192.168.1.0/24 --zone=internal
firewall-cmd --reload        # Apply permanent rules
firewall-cmd --list-all      # View all active rules
⚠️ Without --permanent, changes are lost on reboot. After using --permanent, run --reload to apply immediately.

Network Commands

tracepath hostname            # Trace packet route

Configure Network — nmtui

nmtui    # Launch Terminal UI for network config
# 1. Edit a connection → select interface (e.g., eth0)
# 2. IPv4 Configuration → change Automatic to Manual
# 3. Enter IP address, gateway, DNS
# 4. Back → Set System Hostname
# 5. Activate a connection → deactivate + reactivate

NFS — Network File System

dnf install nfs-utils -y
showmount --exports server_hostname    # Check what server shares

mkdir /mountpoint

# Permanent (fstab):
server:/export  /mountpoint  nfs  rw,sync  0  0
mount -a

AutoFS — On-Demand Mounting

⚠️ Do NOT edit /etc/auto.master directly. Create files inside /etc/auto.master.d/ instead. 
# Direct Mount:
vim /etc/auto.master.d/direct.autofs
# /-    /etc/auto.direct
vim /etc/auto.direct
# /public   -rw,sync   server:/sharing_directory

# Indirect Mount:
vim /etc/auto.master.d/indirect.autofs
# /guest    /etc/auto.indirect
vim /etc/auto.indirect
# mount   -rw,sync   server:/sharing_directory

# Wildcard:
vim /etc/auto.wildcard
# *   -rw,sync   server:/shares/public/&

systemctl restart autofs
CH-16
Podman — Container Management
Images, run, volumes, port forwarding, services, rootless·Advanced
▶

Install & Images

dnf install container-tools     # Install podman
podman search httpd              # Search registries
podman pull IMAGE_URL            # Download image
podman images                    # List local images
podman rmi IMAGE_URL             # Remove image

Containers

podman run -d --name webserver IMAGE_URL
podman ps          # Running containers
podman ps -a       # All containers (running + stopped)
podman stop webserver
podman rm webserver -f    # Force remove

podman exec -it webserver bash      # Enter container shell
podman cp index.html webserver:/var/www/html/

Port Forwarding & Volumes

Port forwarding must be set at container creation — it cannot be added later.
podman run -d --name webserver \
  -p 8080:80 \
  -v /backup/:/var/www/html/:Z \
  IMAGE_URL
# :Z = SELinux context matching

Container as systemd Service (Root)

cd /etc/systemd/system
podman generate systemd --name webserver --files
systemctl daemon-reload
systemctl enable container-webserver.service --now

Rootless Containers

⚠️ SSH directly as the target user. su - username may not behave correctly for rootless container operations.
mkdir -p ~/.config/systemd/user
cd ~/.config/systemd/user
podman generate systemd --name myapp --files
systemctl --user daemon-reload
systemctl --user enable container-myapp.service --now

loginctl enable-linger username    # Start at boot, not just at login

tar, find & Shell Scripts
Archiving, file search, scripting·Intermediate
▶

tar — Archive & Compress

Flag Compression Extension
-j bzip2 .tar.bz2
-z gzip .tar.gz
-J xz .tar.xz 
tar -cvzf backup.tar.gz /usr/local      # Create gzip archive
tar -cvjf backup.tar.bz2 /usr/local     # Create bzip2 archive
tar -xvf archive.tar.gz                 # Extract archive
tar -tvzf backup.tar.gz                 # View contents only

find — Locate Files

find / -name newfile                 # By exact name
find / -user jeel                    # By owner
find / -group devops                 # By group
find / -perm 750 -type d             # Directories with exact perm
find / -perm /g+s                    # Files with SGID set
find / -size +30k -size -45k        # Between 30KB and 45KB

# Execute command on found files:
find / -user jeel -exec cp -pvr {} /root/jeel-files/ \;
# {}  = placeholder  |  \;  = end of exec command

Shell Script Basics

vim myscript.sh

#!/bin/bash
# Shebang — tells system which interpreter to use
find /usr -perm /g+s -size +30k -size -45k > /root/usr-files

chmod +x myscript.sh    # Give execute permission
./myscript.sh           # Run directly
💡 sudo !! re-runs your last command with sudo. !! = shell history expansion for the previous command.

Step 01
Start with the Shell
Everything in Linux begins at the command line. Learn basic navigation, file operations, and shell expansions before anything else.
Step 02
Build a Lab
Set up a local RHEL 9 virtual machine using VirtualBox or VMware. Practice every command in a real environment where mistakes don't matter.
Step 03
Practice Daily
Consistency beats intensity. Spend 30–60 minutes daily practising commands. Use these notes as your reference.
Step 04
Read the Manual
Use man command constantly. Linux manual pages are the most accurate documentation available.
Step 05
Understand the Why
Don't just memorise commands — understand what they do and why. Knowing the concept means you can figure out the syntax even if you forget it.
Step 06
Always Verify
After every configuration change, verify it worked. Run systemctl status and df -h.
